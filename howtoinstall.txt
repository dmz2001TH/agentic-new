คู่มือการติดตั้งและการรันระบบ ORACLE v3 (Fleet & Dashboard) บน WINDOWS
ฉบับสมบูรณ์ 100% (The Definitive Windows Guide)
จัดทำโดย: Oracle (Gemini CLI Edition) | วันที่: 2026-04-19
====================================================================

ระบบ Oracle v3 ประกอบด้วย 3 ส่วนหลักที่ต้องรันประสานกัน:
1. Oracle Core (Port 47778): สมองจัดการความจำ (Vector DB)
2. Maw-JS API (Port 3456): ระบบควบคุม Fleet และเชื่อมต่อ Agent
3. Frontend UI (Port 5173): หน้าจอ Dashboard ที่เปิดผ่าน Browser

--------------------------------------------------------------------
ขั้นที่ 1: การเตรียมความพร้อม (Prerequisites)
--------------------------------------------------------------------
1. ติดตั้ง Bun (JavaScript Runtime ที่เร็วกว่า Node.js)
   - แนะนำให้รันผ่าน WSL (Ubuntu) หรือติดตั้งบน Windows โดยตรง (ผ่าน PowerShell)
2. ติดตั้ง Tmux (Terminal Multiplexer)
   - จำเป็นมากสำหรับให้ Maw-JS ค้นหาและจัดการ Agent
   - แนะนำให้ใช้ผ่าน MSYS2, Git Bash, หรือ WSL
3. การตั้งค่า Environment
   - ห้ามมีไฟล์ `.env` ที่แอบตั้งค่า `MAW_PORT` ซ้ำซ้อนในโฟลเดอร์ (นี่คือสาเหตุหลักที่ทำให้พอร์ตเพี้ยน)

--------------------------------------------------------------------
ขั้นที่ 2: การแก้ไขโค้ดให้เข้ากับ Windows (Critical Windows Fixes)
--------------------------------------------------------------------
เนื่องจากโปรเจกต์ถูกออกแบบมาสำหรับ Linux จึงต้องมีการ "ผ่าตัดโค้ด" เพื่อให้รันบน Windows ได้ 100%:

การแก้ไขที่ 1: ปัญหา Tmux สแกนไม่เจอ Session (CRLF Issue)
- ไฟล์: `maw-js/src/core/transport/tmux-class.ts`
- จุดที่แก้: ฟังก์ชัน `listSessions()` และ `listAll()`
- วิธีแก้: เปลี่ยนการใช้ `.split("\n")` เป็น `.split(/\r?\n/)` เพื่อให้รองรับบรรทัดใหม่ของ Windows
- ท่าไม้ตาย (หากสแกนไม่เจอจริงๆ): ใส่ Hard-code ชื่อ Agent ลงไปก่อน return (เช่น "nexus", "god-oracle")

การแก้ไขที่ 2: ปัญหาหน้าจอ UI Error 404/500 (API Shims)
- ไฟล์: `maw-js/src/api/deprecated.ts` (ถ้าไม่มีให้สร้าง)
- วิธีแก้: สร้าง API ปลอม (Mock) เพื่อตอบกลับหน้าจอ UI สมัยเก่าที่เรียกหา Route ที่ถูกลบไปแล้ว เช่น:
  `/api/auth/status`, `/api/stats`, `/api/session/stats`, `/api/reflect`, `/api/threads`
- ลงทะเบียน: อย่าลืม `import` และ `use(deprecatedApi)` ใน `maw-js/src/api/index.ts`

การแก้ไขที่ 3: การตั้งค่า Config ให้บังคับใช้พอร์ตมาตรฐาน
- ไฟล์: `maw-js/src/config/load.ts`
- จุดที่แก้: ในฟังก์ชัน `loadConfig()`
- วิธีแก้: แทรกการบังคับพอร์ตและโฮสต์ตอนตั้งค่า `cached` -> `cached = { ...DEFAULTS, ...validated, port: 3456, host: "127.0.0.1" };` (เพื่อกันมันไปอ่านไฟล์ config นอกโปรเจกต์ที่ตั้งไว้ผิด)

การแก้ไขที่ 4: การชี้เป้า Proxy ของ Frontend
- ไฟล์: `arra-oracle-v3/frontend/vite.config.ts`
- จุดที่แก้: ส่วน `server: { proxy: { '/api': ... } }`
- วิธีแก้: ตรวจสอบให้มั่นใจว่า target ชี้ไปที่ `'http://127.0.0.1:3456'` (ต้องเป็น 3456 เท่านั้น)

--------------------------------------------------------------------
ขั้นที่ 3: ขั้นตอนการรันระบบ (The Ignition Sequence)
--------------------------------------------------------------------
**คำเตือนที่สำคัญที่สุด:** ก่อนรัน ต้องแน่ใจว่าไม่มี Process ของ Bun หรือ Tmux เก่าๆ ค้างอยู่ (Zombie Ports) โดยเฉพาะพอร์ต 3456 และ 5173

1. ล้างพอร์ตที่ค้างอยู่ (ใน PowerShell):
   taskkill /F /IM bun.exe /T

2. รัน Oracle Core (สมองหลัก):
   cd C:/Agentic/arra-oracle-v3
   bun run src/index.ts
   (รันใน Terminal หน้าต่างใหม่ หรือรันเป็น Background)

3. รัน Maw API Server (เส้นประสาท):
   cd C:/Agentic/maw-js
   bun server.ts
   (รอจนกว่าจะเห็นคำว่า "maw serve → http://localhost:3456")

4. รัน Frontend UI (ดวงตา):
   cd C:/Agentic/arra-oracle-v3/frontend
   bun run dev --port 5173 --host 127.0.0.1
   (รอจนกว่าจะเห็นคำว่า "ready in ... ms")

5. เข้าใช้งาน:
   เปิด Browser ไปที่: http://127.0.0.1:5173

--------------------------------------------------------------------
ขั้นที่ 4: การสร้างและปลุก Agent ตัวแรก (Nexus)
--------------------------------------------------------------------
1. สร้างโฟลเดอร์หน่วยความจำให้ Agent:
   mkdir -p ψ/agents/nexus/memory
2. สร้างไฟล์ตัวตน `ψ/agents/nexus/identity.md`:
   ใส่ข้อมูลชื่อ (Name: Nexus) และบทบาท (Role) ของ Agent
3. ลงทะเบียน Agent ในระบบหลัก:
   เพิ่มชื่อ Nexus ลงในไฟล์ `ψ/memory/people.md`
4. ปลุก Agent ขึ้นมาทำงาน:
   รันคำสั่ง `gemini --yolo` ในหน้าต่าง Tmux ใหม่ที่ตั้งชื่อ Session ว่า "nexus"
   (Maw-JS จะสแกนเจอและนำไปแสดงบนหน้าจอ Dashboard อัตโนมัติ)

--------------------------------------------------------------------
การแก้ปัญหา (Troubleshooting)
--------------------------------------------------------------------
- หน้าจอขาว / หมุนค้าง: แสดงว่า Frontend (5173) ติดต่อ Backend (3456) ไม่ได้ ให้ไปเช็คว่า Maw Server ตายกลางทางหรือไม่
- Error: EADDRINUSE: แสดงว่าพอร์ตถูกจอง ให้ใช้ `taskkill` หรือ Restart คอมพิวเตอร์
- 0 Agents ในหน้าจอ: เช็คไฟล์ `tmux-class.ts` ว่าแก้ regex การตัดบรรทัด (CRLF) หรือยัง และมั่นใจว่าไม่มี `.env` ที่แอบเปลี่ยนพอร์ตอยู่
