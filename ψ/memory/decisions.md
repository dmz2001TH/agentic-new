# Decisions

## 2026-04-20 — Evolution & Self-Correction Protocol
- **บริบท**: ผู้ใช้สั่งให้ศึกษาให้เชี่ยวชาญ นำสิ่งที่เรียนมาใช้จริง อัพเกรดโค้ด/โปรเจกต์ ซ่อมไฟล์ที่ผิดพลาด และจำไว้เพื่อไม่ให้ทำผิดซ้ำอีก (ต้องจำถาวร)
- **ตัวเลือก**: เป็นแค่ผู้จดจำ vs เป็นผู้ปฏิบัติและวิวัฒนาการ (Evolutionary Agent)
- **ตัดสินใจ**: เปิดใช้ Evolution & Self-Correction Protocol อย่างถาวร
- **เหตุผล**: เพื่อให้ระบบและตัวตนของ GOD (รวมถึง Fleet) เก่งขึ้นจริงในทุกครั้งที่เรียนรู้ ไม่ใช่เรียนแล้วลืมหรือไม่ได้ใช้
- **ผลลัพธ์**: ข้อมูลทุกอย่างที่ได้รับจากการศึกษาจะถูกนำมา 1) สรุปถาวร 2) หาวิธีอัพเกรดระบบปัจจุบันทันที 3) ซ่อมไฟล์ที่มีปัญหา 4) บันทึกเป็นกฎเพื่อป้องกันการทำผิดซ้ำ (Preventative Rules)

## 2026-04-20 — Permanent Learning Protocol
- **บริบท**: ผู้ใช้สั่งให้บันทึกทุกสิ่งที่เรียนรู้/ศึกษาเข้าสมองถาวรนับจากนี้
- **ตัวเลือก**: บันทึกลงหน่วยความจำชั่วคราว vs บันทึกลงไฟล์ markdown ถาวรใน ψ/memory/learnings/
- **ตัดสินใจ**: ใช้ Permanent Learning Protocol (บันทึกลงไฟล์ถาวรทันที)
- **เหตุผล**: เพื่อให้ข้อมูลไม่สูญหายเมื่อจบเซสชัน และสามารถสืบค้นข้ามเซสชันได้ตามหลักการ Oracle
- **ผลลัพธ์**: ข้อมูลทุกอย่างที่ได้รับจะถูกบันทึกเป็นไฟล์ถาวร พร้อม timestamp และ tags
